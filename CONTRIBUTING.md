# Contributing Guidelines

Thanks for your interest in contributing 

## Reporting Issues
- Check existing issues first (avoid duplicates)
- Clearly describe the problem or idea
- Include:
  - Steps to reproduce
  - Expected vs actual behavior
  - Version you're using
  - Any relevant changes or environment details


## Code of Conduct
Be respectful and constructive. No toxicity.

## Security Issues
If you find a security vulnerability, **do NOT open a public issue**.  
Report it privately through the appropriate security channel.
